Электронный учебно-методический комплекс Р-423 для платформы Android

Книги и схемы

Программа поддерживает отображение любых документов оформленных с помощью языка разметки HTML и имеющих название index.htm (В случае, если документ имеет иное расширение, например .html, перед помещением в программу нужно всего лишь изменить расширение вручную на требуемое .htm)

Microsoft Word, Microsoft Visio позволяют сохранять и редактировать такие документы.

Файловая структура галереи
// Файловая структура библиотеки книг. Корень - assets 
      books/
              НАЗВАНИЕ_КНИГИ_1/
                              НАЗВАНИЕ_ГЛАВЫ_1/
                                              index.files
                                              index.htm
                              НАЗВАНИЕ_ГЛАВЫ_2/
                                              index.files
                                              index.htm
              НАЗВАНИЕ_КНИГИ_2/
                              index.files
                              index.htm
              ...
  • Каждой книге соответствует папка в books/
  • Если в книге есть разделение на главы, то каждой главе соответствует папка с именем "НАЗВАНИЕ_ГЛАВЫ", внутри папки должен лежать документ с именем index.htm и любые необходимые для работы index.htm папки и файлы
  • Если деления на главы нет, то достаточно просто папки "НАЗВАНИЕ_КНИГИ", внутри которой есть index.htm
  • Kаждый документ может называться только так: index.htm
Алгоритм добавления книги
  1. Есть книга "ИНТЕРЕСНАЯ КНИГА", она состоит из 3х глав, каждая - в отдельном документе word_document.docx
  2. Создаем в папке books/ папку "ИНТЕРЕСНАЯ КНИГА", в ней папки для глав: "Глава 1", "Глава 2", "Глава 3"
  3. Конвертирование в .htm
    Программа Microsoft Word, при сохранении документа (к примеру, пусть он называется index.docx) в формат htm (Сохранить как > > Веб-страница) создаст файл index.htm и вспомогательную папку index.files. Их нужно всегда держать вместе. Т.е. мы помещаем их вместе в папку соответсвующей главы
  4. С помощью Android-Studio пересобираем приложение

Галерея

В галерею можно помещать фотографии, группируя их в смысловые разделы, к каждой фотографіі можно добавіть развернутое описание

Файловая структура галереи
// Файловая структура галереи. Корень - assets 
      gallery/
              descriptions.json
              НАЗВАНИЕ_РАЗДЕЛА_1/
                              название_фото1.jpg
                              название_фото2.jpg
              НАЗВАНИЕ_РАЗДЕЛА_2/
                              название_фото1.jpg
                              название_фото2.jpg
              ...
  • Разделов и фото может быть сколько угодно, но обязательно должен быть хотя бы 1 раздел, в нем хотя бы 1 фото
  • Название раздела и название фотографии отображаются в заголовке при просмотре
Особенности добавления развернутого описания к фотографиям
// Файл descriptions.json 
      {
        "НАЗВАНИЕ_РАЗДЕЛА_1/название_фото1": "Длинное и содержательное описание",
        ...
        "НАЗВАНИЕ_РАЗДЕЛА_3/название_фото45": "Тоже длинное и содержательное описание"
      }
  • Описания фото хранятся в файле descriptions.json со всеми требованиями формата JSON
  • Ключом является "НАЗВАНИЕ_РАЗДЕЛА_1/название_фото1" (Без расширения файла!)
  • Программа сравнивает ключи и путь к фото без учета регистра
  • Значение - строка любой длины
  • Если описания к какой-то фотографии не будет, программа будет работоспособна, просто не покажет описание для этой фотографии
Алгоритм добавления фотографии в галерею
  1. В папке gallery создаем папку для нового раздела (или не создаем, если такой раздел уже есть)
  2. Помещаем фотографию в папку раздела
  3. Если желаем, то в файле descriptions.json, который находится в корне папки gallery/, добавляем строку формата "НАЗВАНИЕ_РАЗДЕЛА/название_фото": "описание описание описание"
  4. С помощью Android-Studio пересобираем приложение
